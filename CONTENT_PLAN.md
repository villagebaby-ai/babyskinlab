## 발행 워크플로 (매일 10시 cron이 수행)

1. 이 캘린더에서 상태가 "대기"인 가장 이른 Day 선택
2. `src/content/guides/{slug}.md` 작성 — frontmatter: title/description(100-150자)/publishedAt(오늘)/keywords/faq(5개+)/relatedTool(해당 축 도구)
3. 본문 5,000자 이상: 도입(문제 공감) → 핵심 답 → 표 1개 이상 → 실전 섹션 → 진료 신호(해당 시) → References 2개 이상(DOI/공식 URL 하이퍼링크)
4. 품질 체크: 존댓말 / en-dash(–) / ruuve 복제 아님 / 내부 링크 2-4개(도구 포함) / "올해·요즘" 시점어 금지
5. 이 파일의 해당 행 상태를 "✅ 발행"으로 갱신
6. `npm run build` 로컬 통과 → push → gh run watch 성공 확인 → 채팅 보고 (URL + Day 번호)
