## 1. 왜 SVG 코드 제작인가 (AI 이미지 생성과 비교)

| 기준 | SVG 코드 제작 ✅ | AI 이미지 생성 (gpt-image-1) |
|---|---|---|
| 스타일 일관성 | 완벽 (같은 팔레트·선·폰트) | 매번 미묘하게 다름 |
| 한글 텍스트 | 완벽 렌더 | 한글 깨짐·오타 사고 잦음 |
| 파일 크기 | 5-15KB (Core Web Vitals 유리) | 200KB-1MB |
| 비용 | 0원 | 장당 과금 |
| 수정 | 텍스트 수정으로 즉시 | 재생성 필요 |
| 브랜드 룩 | 클리니컬 랩과 정확히 일치 | 프롬프트 복불복 |

과학 도해 + 심플 = SVG가 정답. 사진이 필요해지면(제품·인물) 그때 별도 논의.

## 2. 콘텐츠당 2장 구성 공식

| 이미지 | 역할 | 위치 | 예시 (Day 1) |
|---|---|---|---|
| **① 원리도** | 글의 핵심 개념을 한 장으로 | 첫 번째 핵심 섹션 뒤 | 로션·크림·연고 수분:유분 비율 비교 |
| **② 실전도** | 행동 가이드·타임라인·체크 | 실전 섹션 뒤 | 목욕 후 3분 보습 골든타임 타임라인 |

원리도는 "왜"를, 실전도는 "어떻게"를 그린다. 장식용 이미지 금지 — 본문 없이 이미지만 봐도 정보가 전달되어야 한다.

## 3. 스타일 규격 (모든 다이어그램 공통)

- **비율·크기**: viewBox `0 0 1200 800` (3:2 가로)
- **배경**: 흰색 `#ffffff`, 테두리 없음
- **팔레트** (5색 이내):
  - 주선·강조: teal `#0ea5a4` / 진한 강조 `#0b8483`
  - 보조 채움: teal pale `#e6f7f7`, amber pale `#fef3c7` (대비 필요시)
  - 텍스트: ink `#0f172a`, 보조 텍스트 `#64748b`
  - 위험·주의 표시만 `#ef4444` 허용
- **선**: 굵기 3px, 둥근 끝(`stroke-linecap="round"`), 모서리 radius 12-16
- **타이포**: `font-family="Pretendard, 'Apple SD Gothic Neo', sans-serif"` — 제목 34-38px bold, 라벨 22-26px, 캡션 18-20px
- **레이아웃**: 상단 제목 1줄 → 중앙 도해 → 하단 라벨. 여백 넉넉히 (콘텐츠는 안전영역 1100×700 안에)
- **금지**: 그라데이션 남용, 3D, 클립아트 감성, 이모지 남발, 촘촘한 텍스트
- **워터마크**: 우하단 `babyskinlab.com` 18px `#cbd5e1` (옅은 로고 텍스트만 — 큰 텍스트 워터마크 금지)

## 4. 파일·SEO 규칙 (이미지 SEO 7체크)

1. 파일: `public/diagrams/{글-slug}-1.svg`, `-2.svg` (slug와 파일명 일치)
2. 본문 삽입: `<figure>` + `<img loading="lazy" width="1200" height="800">` + `<figcaption>` — 캡션은 이미지 설명이 아니라 **본문 takeaway 사실 한 줄**
3. `alt`: 도해가 전달하는 정보를 문장으로 (예: "로션 크림 연고의 수분 유분 비율 비교 — 로션 7:3, 크림 5:5, 연고 2:8")
4. frontmatter `images:` 배열 → Article 스키마 `image` 자동 출력
5. sitemap `image:image` 자동 출력 (astro.config가 public/diagrams 스캔)
6. 표시 폭: figure max-width 640px 중앙 정렬 (본문 대비 60-75%)
7. width/height 속성 명시 (CLS 방지)

## 5. 제작 워크플로 (매일 발행 cron에 포함)

1. 글 작성 후 원리도·실전도 주제 결정 (2절 공식)
2. SVG 2장 코드 작성 → `public/diagrams/`
3. 본문에 figure 2개 삽입 + frontmatter `images` 배열 기입
4. 빌드 후 시각 확인 (텍스트 겹침·잘림 체크)
